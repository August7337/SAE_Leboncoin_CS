
## Dotnet ef

0. Installation
```
dotnet tool install --global dotnet-ef --version 8.0.11
```

###1. Supprime la migration précédente (celle qui a généré le mauvais script) :
```
dotnet ef migrations remove --project "TP4 partie 1"
```
Note : Si la suppression échoue car la migration a déjà été appliquée (partiellement), tu devras peut-être supprimer la base de données manuellement dans PgAdmin ou faire dotnet ef database update 0 --project "TP4 partie 1" avant le remove.

###2. Recrée la migration :
```
dotnet ef migrations add CreationBDFilmRatings --project "TP4 partie 1"
```

###3. Vérifie le script :
```
dotnet ef migrations script --project "TP4 partie 1"
```
Tu devrais maintenant voir apparaître GENERATED BY DEFAULT AS IDENTITY, DEFAULT 'France', DEFAULT (now()), UNIQUE INDEX, et ON DELETE RESTRICT.

###4. Applique à la base :
```
dotnet ef database update --project "TP4 partie 1"
```

```
dotnet ef database
```

###5. Générer le contrôleur
```
dotnet aspnet-codegenerator controller -name UtilisateursController -async -api -m Utilisateur -dc FilmRatingsDBContext -outDir Controllers
```
